HTML

  • The head element contains information about the webpage.
  • The body element represents the visible content shown to the user.
  • The "alt attribute" is important for screen readers and other assistive tecnologies.
  • The h1 element should only appear once per page.
  • HTML elements are case sensitive.
  • Child elements are elements nested inside other elements.
  • Parent elements are elements that hold children elements.
  • Sementic HTML elements - the name of the element descrives its content and the role it plays on the webpage.
  • Non-semantic HTML elements include "div" (division in the page) and "span" (inline division).
  • Images (img) contain attributes like src, which defines the location of the image.

CSS

  • A margin indicates how much space we want around the outside of an element.
  • A padding indicates how much space we want around the content inside an element.
  • Selector defines the element or attributes to which the rules, or declarations, will apply.
  • Declarations are the CSS property we want to apply and the value of the property.
  • Inline CSS - style directly in the HTML file. This is good for quick, permanent changes. Use "style" attribute.
  • Internal CSS Style Sheet - embedded style sheet in the HTML file. Use "style" in the "head" element. It only applies to that page.
  • External CSS Style Sheet - This is the most common. It makes design changes easier. Links to the HTML file using "link" in the "head" element.

Git

  • git status: Checks what branch we are currently on.
  • git checkout -b branch-name: Creates a new branch and switches to it.
  • git checkout main: Moves back to the main branch.
  • Git commands are not the same as bash commands and will always start with "git."
  • git add -A: Adds all modifications in the current working branch to the staging area.
  • git commit -m "comments": Associates a message with the commit.
  • git pull origin main: Receives the branches modifications into the local environment.
  • git push origin feature/feature_name: This moves the changes to the remote GitHub branch.

JavaScript

  • JavaScript is a programming language that can add interactivity to a website.
  • A variable is a named container that allows us to store data in our code.
  • Control flow is the order in which a computer executes code in a script.
  • A function is a set of instructions that tells the computer how to perform a certain task.
  • You can cure Math.floor() and Math.random() to products a random number/variable.
  • JavaScript is case sensitive.
  • An array is a group of related data and is not limited to a single type of data.
